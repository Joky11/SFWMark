# Log 文件状态分析指南

## 读取策略

- 先读取 log 文件的最后 50 行，了解当前运行到哪一步
- 再读取开头 30 行，了解启动参数和配置
- 如果需要更多上下文，有针对性地搜索关键信息（如 loss 值、epoch 进度、warning 等）
- 不要全量加载 log 文件

## 分析内容

### 1. 当前状态
- 程序运行到了哪一步（第几个 epoch、第几个 batch、哪个阶段）
- 当前的关键指标（loss、accuracy、学习率等）
- 预估剩余时间和进度百分比

### 2. 趋势判断
- loss 是否在正常下降，有没有震荡或发散的迹象
- 如果有多个 epoch 的数据，判断训练是否收敛
- 学习率调度是否正常

### 3. 潜在问题预警
- Warning 信息汇总（特别是 deprecation、数值溢出、梯度相关的警告）
- GPU 显存使用趋势（是否有 OOM 风险）
- 异常慢的 batch 或 step（可能的 IO 瓶颈或数据加载问题）
- loss 出现 NaN 或 Inf 的风险信号
- 重复出现的非致命错误

### 4. 配置审查
- 检查启动参数是否合理（batch_size、learning_rate、num_workers 等）
- 数据集路径和模型路径是否正确加载
- 随机种子是否设置（可复现性）

## 输出格式

- 用简洁的方式汇报：当前状态 → 趋势 → 潜在问题
- 如果一切正常，直接说"运行正常，暂无问题"
- 如果发现风险，按严重程度排序列出
